## 阶段 9 完整验收

### 物化索引 1.5

阶段 9 不再让 Word List 在每次请求中扫描并聚合 Token 表。加工阶段生成：

```text
kwic_index.sqlite
├── tokens
├── ngrams
├── parallel_pairs
├── word_totals
└── word_frequencies
```

`word_totals` 保存语言、规范词形、展示词形、Frequency、Document Range 和标点标志；`word_frequencies` 额外按 POS 物化。Word List、Keyword、Wordcloud 共享这些预计算事实表。1.5 将未标注中文从逐字正则升级为 Jieba 0.42.1；旧索引不会在 Web 请求中静默回填，必须离线重建，防止同一页面混用字级和词级统计。

分词口径按语言记录：原始中文为 `zh:jieba-0.42.1`，原始英文为 `en:regex-baseline-v1`，中英原始对齐语料同时记录两侧口径；老师带 POS 的语料为 `source-provided-pos-v1`。ParaConc 的 `parallel_pairs` 仍由老师段落顺序或 `p@n/s@n` 编号生成，中文分词升级不参与重建对齐。

### Keyword

页面：

```text
/statistics/<target_corpus_id>/keywords/
```

支持：

- 从当前用户可访问且 ready 的语料中选择参照语料；
- 所选语言必须同时存在于目标和参照语料；
- 任一语料最小 Frequency、最小 Document Range；
- 词项过滤、标点开关、负关键词开关；
- Log-Likelihood、Chi-square、绝对 Log Ratio、目标频次和词项排序；
- 目标/参照 Frequency、Range、Per million 并排显示；
- 结果分页，点击词项跳转目标语料 KWIC；
- 私有参照语料不会进入选择列表，也不会泄露名称。

统计采用四格表：

```text
                 Item       Other
Target           O11        O12
Reference        O21        O22

Log-Likelihood = 2 × Σ O × ln(O / E)
Chi-square      = Σ (O - E)² / E
Log Ratio       = log2((O11 / Target Tokens) / (O21 / Reference Tokens))
```

零频采用 `0.5` 修正计算 Log Ratio。正值表示目标语料相对高频，负值表示目标语料相对低频。

黄金答案：

```text
Target:    apple apple apple banana   (4 Tokens)
Reference: apple banana banana banana (4 Tokens)

apple:
  Target Freq=3, Reference Freq=1
  LL=2.0929925750581915
  Chi-square=2.0
  Log Ratio=1.584962500721156

banana:
  Target Freq=1, Reference Freq=3
  Log Ratio=-1.584962500721156
```

黄金文件：`backend/tests/expected/statistics/keyword.json`。

### Wordcloud

页面：

```text
/statistics/<corpus_id>/wordcloud/
```

支持：

- 基于 `word_totals`，不扫描原文；
- 语言、最小 Frequency、最大 25/50/100/200 词；
- 空格、逗号、中文标点或换行分隔停用词；
- 默认排除标点，可显式包含；
- Frequency 对数缩放为 18–72 px；
- 深海蓝、森林绿、落日橙三套主题；
- 响应式 SVG、确定性椭圆螺旋布局和碰撞检测；
- 悬停显示原始频次，点击进入 KWIC；
- 语义化 `aria-label`，无结果时有明确空状态。

自动测试验证 `apple=3` 显示 72 px、`banana=1` 显示 18 px，所有词项坐标位于 1000×560 画布内；加入停用词 `apple` 后仅保留 `banana`。页面测试同时验证 SVG、主题和防重叠布局标识。

## 阶段 10 完整验收

### 模块边界

```text
apps/search/query_parser.py  只负责语法扫描、验证和 QueryPlan
apps/search/filters.py       只负责 TokenFilter 与参数化 SQL 条件
apps/search/query_engine.py  只负责连续 Token 匹配、排序和分页
apps/search/views.py         只负责权限、表单和引擎路由
```

复杂查询复用 `KwicHit`、左右窗口、L1/L2/L3/R1/R2/R3、元数据读取和结果模板。普通 KWIC 仍走原 `KwicSearchEngine`，因此复杂语法没有改变已有精确词/短语语义。

### 支持语法

| 类型 | 示例 | 语义 |
|---|---|---|
| 精确词 | `development` | 一个规范化 word Token |
| 精确短语 | `"high quality"` | 同一句内连续 Token |
| 通配符 | `develop*`、`?ocus` | `*` 任意长度，`?` 单字符 |
| starts_with | `starts_with(develop)` | word 前缀 |
| ends_with | `ends_with(ment)` | word 后缀 |
| contains | `contains(velo)` | word 子串 |
| Token 属性 | `[word="development"]`、`[pos="NN1"]`、`[lemma="develop"]` | 属性精确或通配符匹配 |
| 属性序列 | `[pos="JJ"] [lemma="economy"]` | 连续 Token 条件组合 |

最多 20 个 Token 条件，单表达式最多 500 字符，单个值最多 100 字符。当前语料如果没有独立 lemma 标注，lemma 查询明确回退到规范化 word；一旦索引含 lemma 值则优先查询真实 lemma。该回退保证功能可用，但不能冒充语言学词形还原。

### 安全约束

- 属性只允许 `word/lemma/pos`，函数只允许 `starts_with/ends_with/contains`；
- SQL alias 和属性列来自代码白名单；所有查询值均为 SQLite 参数，不参与 SQL 拼接；
- `LIKE` 中 `%`、`_`、反斜线先转义，用户通配符仅转换 `*` 和 `?`；
- 拒绝控制字符、分号、未闭合引号/括号、未知函数/属性和纯通配符；
- 查询仍先通过 `visible_corpora_for()`，不能借复杂表达式绕过语料权限；
- 不开放任意 CQP 命令、宏、标签、全局约束或文件写入。

## 与官方工具的当前差异

### AntConc

当前阶段已达到开发手册要求的六个子模块，但没有声称完全复制 AntConc 4.4.1：

- Keyword 当前比较 Word 词表；AntConc 还可比较 Cluster、N-Gram、Collocate 来源；
- 已提供 LL、Chi-square、Log Ratio，尚未提供 p 值阈值、Bonferroni 和更多效应量；
- Wordcloud 当前来源是 Word List，已支持 SVG 螺旋布局和主题；尚未接入 Scratchpad、Keyword/N-Gram/Collocate 输出、遮罩和 PNG 导出；
- 页面以正确、可访问和可分页为先，没有复制桌面端布局/快捷键。

### ParaConc

当前平台保留人工段落顺序及 `p@n/s@n` 编号，支持中英互查、AND/NOT、标签/POS 索引和句/段视图。尚未实现官方说明中的正则平行搜索、半自动对齐编辑器、Hot Words 和高级多级排序。

### CQPweb / CQP

当前实现是面向现有 SQLite Token 索引的安全 CQP 风格子集，不声称兼容完整 CQP：

- 已支持开发手册要求的 word/phrase/wildcard/starts/ends/contains/POS/lemma；
- 尚未支持布尔表达式、Token 可选/重复、标签、全局 `::` 约束、结构属性、宏和保存命名查询；
- 当前源文件没有独立 lemma 时回退到 normalized word，研究级 lemma 查询需要未来导入器接收真实 lemma 标注；
- 语法错误在表单中返回中文定位信息，不把 SQLite 或 Python 异常暴露给用户。

## 人工验收步骤

1. 登录测试账号，打开标注双语语料的 AntConc Tools。
2. 在 Word List 中选择中文和 POS，核对 Frequency 与 Per million。
3. 在 N-Gram 中查询“农民协会”，确认不跨句。
4. 在 Collocates 中输入“农民”，调整 L/R span，核对 LogDice。
5. 打开单语中文《中国社会各阶级的分析》的 Keywords，选择双语人工段落对齐语料为参照，设置中文、最小频次 2。
6. 确认出现“资产阶级、无产阶级、阶级”等多字词，Target/Reference、LL、Chi²、Log Ratio 同时显示；打开标注语料的 Keywords，确认不兼容的 Jieba 参照不会出现。
7. 打开“显示负关键词”，确认出现负方向记录；点击词项进入 KWIC。
8. 在 Wordcloud 中设置最小频次 10、最大 25 词，加入停用词“的 了 是”。
9. 选择三套配色并调整窗口宽度，确认 SVG 无重叠且“农民”为最大词项；悬停显示 259，点击进入 KWIC。
10. 回到 ParaConc 查询“第三件”，确认中英人工句对未变化。
11. 使用无权账号尝试私有目标/参照语料，确认 403 或参照项不可见。
12. 在 KWIC 选择“CQP 子集”和中文，依次查询 `资产*`、`contains(阶级)`，核对 18/64 条命中。
13. 打开标注双语语料，选择英文，查询 `[pos="NN1"]` 和 `[lemma="peasant"]`。
14. 输入未闭合引号及未知属性，确认页面显示中文错误且服务日志没有 500。
15. 将每页设为 1 并选择 R1，确认翻页链接保留 `query_mode`、`language` 和表达式。
